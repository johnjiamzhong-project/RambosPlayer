每个按钮背后发生了什么

📄 打开文件 用户操作
点击"打开" → QFileDialog → MainWindow::openFile() → PlayerController::open(path)
  1. QFileDialog 弹出系统文件选择器，用户选一个媒体文件
  2. PlayerController::open() 先调 stop() 停止一切旧播放，重置三条队列
  3. DemuxThread::open() 执行 avformat_open_input 打开容器，avformat_find_stream_info 探测流，找到视频/音频流索引
  4. 读取流参数（分辨率、采样率、时间基），依次初始化 VideoDecodeThread、AudioDecodeThread、VideoRenderer
  5. 发射 durationChanged 信号 → 界面更新进度条范围
  • 此时只初始化，线程还没 start，不占用 CPU
    • ▶️ 播放 / 暂停 用户操作

    点击播放

    点击播放按钮 → PlayerController::play() → start 四个组件
    1. 检查 playing_ 标志防止重复播放
    2. DemuxThread::start() → 线程开始 av_read_frame 循环，读包推入两条 packet 队列
    3. VideoDecodeThread::start() → 线程开始从 videoPacketQ 取包，解码后推入 videoFrameQ
    4. AudioDecodeThread::start() → 线程开始从 audioPacketQ 取包，解码重采样→QAudioOutput 出声，同时更新 AVSync 时钟
    5. VideoRenderer::startRendering() → 启动 1ms QTimer，逐帧取 videoFrameQ，sws_scale 转换后 QPainter 绘制
    6. posTimer_ 启动（100ms）→ 轮询 AVSync 音频时钟 → positionChanged 信号 → 进度条走动

    暂停时

    • posTimer_ 停止，进度条不动
    • QAudioOutput::suspend()，扬声器静音
    • 渲染 1ms 定时器停止，画面不动
    • 解码线程继续跑，队列逐渐积压
    • 恢复播放时 resume QAudioOutput + 重启定时器

    关键设计

    • 暂停不销毁线程，避免重建开销
    • 队列积压提供即时恢复的缓冲
    • 音频钟停在暂停前位置，恢复后连续
    ⏱️ 拖拽进度条 用户操作
    拖拽滑块 → onSeekSliderMoved → PlayerController::seek(s)
    1. sync_.setAudioClock(seconds) — 先把时钟跳到目标位置，VideoRenderer 看到旧帧 pts 远小于时钟 → 判定落后 400ms → 丢弃旧帧，避免队列堵满
    2. renderer_->flushPendingFrame() — 丢弃渲染器暂存的那一帧
    3. demux_.seek(seconds) — 原子写入 seekTarget_，DemuxThread::run() 下次循环执行 av_seek_frame 跳到关键帧附近，然后清空两条 packet 队列（pop + free）
    4. videoDec_.flush() / audioDec_.flush() — 清空解码器内部缓冲（avcodec_flush_buffers），避免残留旧数据
    5. 解码线程从新位置开始解码 → 渲染新画面、输出新声音
  • 先更新音频钟是 关键：否则旧帧不会被丢弃，videoFrameQ 堵满 → DemuxThread 的 push 卡在队列上 → seek 无法执行
  • 清队列用 pop+free（不是 clear()），因为 clear 不释放 AVPacket* 内存，会泄漏
  • av_seek_frame 使用 AVSEEK_FLAG_BACKWARD 跳到最近的关键帧，不是精确到秒
    • 🔊 音量控制 用户操作
    拖拽音量滑块 → onVolumeChanged(value) → PlayerController::setVolume(v) → AudioDecodeThread::setVolume(v)

    内部机制

    • 音量值 0–100 → 转为 0.0–1.0 float
    • 写入 AudioDecodeThread 的原子量 pendingVolume_
    • 音频线程每次写入前检查 pendingVolume_，若改变则设置 QAudioOutput 音量
    • 原子量无锁，不阻塞音频渲染

    界面细节

    • 音量滑块用 eventFilter 拦截 mousePressEvent + mouseMoveEvent
    • 点击任意位置直接跳转，无需拖拽
    • 滑块上方有静音/非静音图标切换
    🛠 总览：界面信号 → 数据层
    点击播放 → play() → DemuxThread::start() + VideoDecodeThread::start() + AudioDecodeThread::start() + VideoRenderer::startRendering() + posTimer_
    点击暂停 → pause() → posTimer_.stop() + VideoRenderer::stopRendering() + AudioDecodeThread::setPaused(true)
    拖拽进度 → seek(s) → setAudioClock(s) + DemuxThread::seek(s) + VideoDecodeThread::flush() + AudioDecodeThread::flush()
    调节音量 → setVolume(v) → AudioDecodeThread::atomic store → QAudioOutput::setVolume()
    播放结束 → DemuxThread EOF → onDemuxFinished() → 500ms 延迟 → playbackFinished → 界面复位
