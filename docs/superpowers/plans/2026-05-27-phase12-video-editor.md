## Part 1 — 三种剪切模式

三种模式共享 Timeline 控件，区别在于如何确定入点/出点。

### 1.1 自由剪切（已完成）

**入口：** 菜单栏"剪辑 → 自由剪切 (Ctrl+T)"
**交互：** 拖拽 Timeline 左右把手设置入点/出点
**输出：** 导出为单个片段
**状态：** ✅ Phase 10 已完成

### 1.2 浏览剪切（新建）

**入口：** 菜单栏"剪辑 → 浏览剪切 (Ctrl+B)"
**交互流程：**
1. 开启浏览剪切模式
2. 视频继续/开始播放
3. 用户按**设置入点键**（默认空格）→ 当前播放时间标记为入点，在底部导轨上标记绿色定位点
4. 用户按**设置出点键**（再次按空格）→ 标记出点，底部导轨显示红色定位点
5. 一个片段标记完成，在底部导轨生成一个高亮区间块
6. 重复步骤 3-5 标记更多区间
7. 关闭浏览剪切模式 → 弹出确认对话框，选择保留的区间段，确定后自动填充到 Timeline

**关键行为：**
- 播放/暂停功能与剪切标记互不影响。播放中按空格同时执行"暂停+标记"还是"仅标记不暂停"？设计为：浏览模式下，空格只做标记，视频持续播放
- 底部导轨是 Timeline 下方的辅助条，专门显示已标记的区间段（多个），区别于 Timeline 本身的单对把手
- 暂停时在浏览模式下询问是否退出（"退出浏览剪切？标记的区间将保留"）

### 1.3 多段剪切（新建，题主可自命名，暂用此名）

**入口：** 菜单栏"剪辑 → 多段剪切 (Ctrl+M)"
**交互流程：**
1. 开启多段剪切模式
2. 弹出输入面板（嵌入底部的 QWidget 或独立对话框）
3. 用户逐行输入区间：`00:10 - 00:25`、`01:30 - 02:00` ...
4. 每次输入一行，自动追加到列表
5. 用户点击"验证" → 检查每个区间：
   - 起始 ≤ 结束
   - 结束 ≤ 视频总时长
   - 区间之间不重叠（或允许重叠？由用户决定）
6. 验证通过后确定 → 区间列表同步显示在 Timeline 底部导轨

## Part 2 — 合并功能

### 2.1 视频拼接（Video Concat）

| 条件 | 方案 | 性能 |
|------|------|------|
| 所有输入编码参数完全一致 | concat demuxer + `-c copy` | 不重编码 |
| 输入参数不同 | concat filter + 重编码 | CPU 重编码 |

**自动判断逻辑：**
1. 读取所有输入文件的 `codecpar`（codec_id、width、height、pix_fmt、sample_rate）
2. 全部一致 → concat demuxer；任一不一致 → concat filter

### 2.2 音频混音（Audio Mix）

- amix filter：`amix=inputs=N:duration=first:dropout_transition=2`
- 每路输入可调音量权重
- 输出时长策略：取最长输入

### 2.3 音视频混流（Mux）

- 独立视频文件 + 独立音频文件
- 分别 `av_read_frame`，按 PTS 交错写入
- 视频已有音轨时可选择替换

## 3. Timeline 改造

Timeline 现有模式（入点/出点把手）保持不变，同时支持扩展的**底部导轨层**：

```
┌─────────────────────────────────────────────┐
│  时间刻度尺                                   │
├─────────────────────────────────────────────┤
│  缩略图轨道                                   │
│  ┌──────────┬────────────────┬──────────┐   │
│  │▸ in      │  高亮区域      │          │   │
│  │  (绿色)  │  (蓝)         │ out (红) │   │
│  └──────────┴────────────────┴──────────┘   │
├─────────────────────────────────────────────┤
│  底部导轨（浏览/多段模式下的多区间显示）         │
│  ████████         ████████████               │
│  00:10-00:25      01:30-02:00               │
└─────────────────────────────────────────────┘
```

**新增接口：**
- `addSegment(int64_t startUs, int64_t endUs)` — 添加一个区间段到底部导轨
- `clearSegments()` — 清空所有区间段
- `segments()` → `QList<QPair<int64_t,int64_t>>` — 获取所有区间
- `setBottomBarVisible(bool)` — 显示/隐藏底部导轨

## 4. 实现顺序

### Stage A — Timeline 底部导轨扩展
- Timeline 新增底部导轨层绘制
- `addSegment()` / `clearSegments()` / `segments()`
- 单元测试：导轨绘制宽度与时长成比例、多段不重叠

### Stage B — 浏览剪切控制器
- `BrowseClipper` 连接播放器状态 + 键盘/按钮事件
- 标记入点/出点 → 追加到 Timeline 底部导轨
- 退出浏览时弹出确认对话框
- 手动测试：播放中标记 3 段，退出，查看导轨显示

### Stage C — 多段剪切控制器 ✅
- `SegmentClipper` + 输入面板 UI（QDialog：QTextEdit + 验证列表）
- 时间格式解析（MM:SS / HH:MM:SS，分隔符支持 `-` `→` `,`）
- 验证：时长上限、起始≤结束、区间不重叠
- 确定后填充 Timeline 底部导轨（只追加，不清除已有区间）
- 三种剪切模式互斥逻辑（进入/退出/取消时自动切换）
- 手动测试：输入 3 段，验证正常 ✅

### Stage D — 合并：同参数视频拼接（Stage D1）+ 音频混音（Stage D2）+ 异参数拼接（Stage D3）+ 音视频混流（Stage D4）
- 参见 `docs/MERGE-PLAN.md`（已废弃，内容并入本文件）

### Stage E — MergePanel UI + MainWindow 集成
- 拖入文件列表
- 模式自动判断
- 各文件音量滑块
- 开始/取消、进度条、日志

## 5. 验收标准

| 场景 | 预期 |
|------|------|
| 浏览剪切：播放中标记 3 段 | 退出后 Timeline 显示 3 个区间，可选择保留 |
| 多段剪切：输入 `00:10-00:20`, `01:00-01:30` | 验证通过，底部导轨显示对应区间 |
| 自由剪切：拖拽把手导出 | 输出文件时长与选区一致 |
| 拼接 2 个同参数 mp4 | 输出时长 = 两者之和，音画同步 |
| 原声 + BGM 混音 | 两路声音同时可听 |
| 无声视频 + 独立音频 合成 | 输出文件同时含音视频 |
