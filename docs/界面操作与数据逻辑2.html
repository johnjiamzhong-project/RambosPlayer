界面操作 → 数据逻辑

四个核心操作：打开文件、播放暂停、拖拽进度、调节音量

1 打开文件
点击"打开"→ MainWindow→ PC::open(path)→ DemuxThread::open→ avformat_open_input + find_stream_info

解复用线程做了什么

  • avformat_open_input — 打开容器文件
  • avformat_find_stream_info — 探测流参数
  • 遍历流，找到 videoIdx / audioIdx
  • 保存 duration（微秒）

控制器接着做了什么

  • 读视频流参数 → videoDec_.init()
  • 读音频流参数 → audioDec_.init()
  • renderer_->init(w, h, timeBase, &sync_, &queue)
  • emit durationChanged → UI 更新进度条范围
此时只初始化，线程尚未 start，不占 CPU
2 播放 / 暂停
▶ 播放→ PC::play()→ Demux+ VideoDec+ AudioDec+ Renderer+ posTimer
⏸ 暂停→ PC::pause()→ 停止渲染定时器+ 音频 suspend+ posTimer 停

播放时启动了什么

  • DemuxThread 开始 av_read_frame 循环
  • VideoDecodeThread 开始解码
  • AudioDecodeThread 开始解码 + 输出 PCM
  • VideoRenderer 1ms 定时器开始拉帧
  • posTimer 100ms 开始发出位置更新

暂停 vs 停止

  • 暂停：线程不退出，音频 suspend
  • 停止：stop + wait 等待线程退出
  • 暂停恢复快（no-op start + resume）
  • 停止后重播需重新初始化线程
暂停不销毁线程，解码线程继续跑但队列积压；恢复时 resume QAudioOutput + 重启定时器
3 拖拽进度条 (Seek)
拖拽滑块→ onSeekSliderMoved→ PC::seek(s)
① sync_.setAudioClock(s) 先更新音频钟到目标位置，让 Renderer 丢弃旧帧
② renderer_ flushPendingFrame 丢弃渲染器暂存的那一帧
③ demux_.seek(s) 原子写 seekTarget_ → run() 顶部 handleSeek() → av_seek_frame + 清空双队列
④ videoDec_.flush() audioDec_.flush() 清空解码器内部缓冲
① 必须在 ③ 之前执行：否则旧帧不被丢弃 → videoFrameQ 堵满 → DemuxThread push 阻塞 → seek 永远无法执行
4 音量控制
拖动音量滑块→ onVolumeChanged(0-100)→ setVolume(0.0-1.0)→ AudioDec::setVolume→ QAudioOutput 音量改变

实现方式

  • 滑块值 0–100 转为 0.0–1.0
  • 写入 AudioDecodeThread 的原子量
  • 音频线程下个循环检查到变化
  • 调 QAudioOutput::setVolume()

界面细节

  • eventFilter 拦截点击直接跳转
  • 滑块图标随静音/非静音切换
  • 原子操作无锁，不阻塞音频
音量是最简单的操作 — 只改一个原子量，不涉及任何线程同步或队列操作

四个操作执行顺序

打开文件 init 全部组件
↓ 必须先打开文件
播放 Demux start → VideoDec start → AudioDec start → Renderer start
↓ 播放过程中
