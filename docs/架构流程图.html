完整数据流路径

  1. 打开文件 — MainWindow 选择文件 → PlayerController::open(path) → 创建 DemuxThread → 传入 videoPacketQ / audioPacketQ 两个队列
  2. DemuxThread::open() — avformat_open_input 打开容器 → avformat_find_stream_info 探测流参数 → 遍历流找到 videoIdx / audioIdx → 保存 duration
  3. 启动流水线 — PlayerController::play() → 依次 start DemuxThread / VideoDecodeThread / AudioDecodeThread → VideoRenderer::startRendering() 启动 1ms Timer
  4. 解复用循环 — DemuxThread::run() → av_read_frame 读取一个 AVPacket → 判断 stream_index → push 到 videoPacketQ 或 audioPacketQ
  5. 视频解码 — VideoDecodeThread::run() → tryPop videoPacketQ → avcodec_send_packet → avcodec_receive_frame → push AVFrame* 到 videoFrameQ
  6. 音频解码 — AudioDecodeThread::run() → tryPop audioPacketQ → avcodec_send_packet → avcodec_receive_frame → swr_convert 重采样 → write 到 QAudioOutput → setAudioClock(pts) 更新主时钟
  7. 视频渲染 — VideoRenderer::onTimer() (1ms) → tryPop videoFrameQ → AVSync::videoDelay(pts) 判断等待/丢弃/渲染 → sws_scale YUV→RGB → update() → paintEvent() QPainter 居中绘制
  8. 位置同步 — PlayerController 100ms QTimer → 轮询 AVSync::audioClock() → emit positionChanged(ms) → MainWindow 更新进度条
  9. Seek 跳转 — MainWindow 拖拽进度条 → PlayerController::seek(s) → DemuxThread::seek(s) 原子存储目标 → run() 检测后 av_seek_frame → 清空两个 packet 队列 → 解码线程 flush() 清空缓冲区
  10. 播放结束 — DemuxThread run() 读到 EOF → emit demuxFinished → PlayerController::onDemuxFinished() → 等待队列消费完 → emit playbackFinished → MainWindow 复位 UI

核心模块说明

FrameQueue<T> 有界阻塞队列

线程安全的生产者-消费者队列。满时 push() 阻塞等待，空时 tryPop(timeout) 阻塞等待。
abort() 立即唤醒所有等待线程（退出信号）。clear() 清空队列（seek 用）。
三个实例：videoPacketQ(100) / audioPacketQ(400) / videoFrameQ(15)

AVSync 音频主时钟

std::atomic<double> 存储音频 PTS（秒），无锁读写。
AudioDecodeThread 写入 → VideoRenderer 通过 videoDelay(pts) 查询等待时间。
视频落后超过 400ms 返回 0（直接丢帧），领先则返回延迟毫秒数。

DemuxThread QThread

独立线程中循环 av_read_frame，按 stream_index 分发 AVPacket* 到视频/音频队列。
seek() 原子存目标秒数，run() 下次循环检测并 av_seek_frame，然后清空双队列。
abort 标志 + 队列 abort() 唤醒保证安全退出。

VideoDecodeThread QThread

消费 videoPacketQ → avcodec_send_packet / receive_frame → push AVFrame* 到 videoFrameQ。
flush() 在 seek 后清空解码器内部缓冲，避免残留旧帧导致画面错乱。

AudioDecodeThread QThread

消费 audioPacketQ → 解码 → swr_convert 重采样为 S16 Stereo 44100Hz → QAudioOutput 输出。
每次写入后调用 AVSync::setAudioClock(pts) 更新主时钟。
支持 pause/resume (suspend QAudioOutput)、volume 调节。

VideoRenderer QWidget

1ms QTimer 驱动 → tryPop videoFrameQ → AVSync::videoDelay() 决定等待/丢弃 →
sws_scale YUV420P → RGB32 → QImage → update() → paintEvent 中 QPainter 保持宽高比居中绘制。
未到渲染时间的帧暂存 pendingFrame_，避免推回队列阻塞解码线程。

PlayerController QObject

播放器总控制器，持有所有组件（DemuxThread, VideoDecodeThread, AudioDecodeThread, VideoRenderer, AVSync, 三队列, posTimer）。
对外暴露 open/play/pause/stop/seek/setVolume 统一接口。
100ms QTimer 轮询 AVSync 发出 positionChanged 信号。

MainWindow QMainWindow

主窗口，UI 全部在 .ui 文件中用 Qt Designer 绘制。包含 VideoRenderer (promoted widget)、进度条、播放/暂停按钮、音量滑块、最近文件菜单。
键盘快捷键、双击全屏、eventFilter 处理音量滑块点击。

跨线程通信：FrameQueue 是唯一通道

设计原则

解码线程之间不使用 Qt signal/slot 直接通信，所有数据传递都通过 FrameQueue。 这样避免了跨线程信号连接的复杂性，队列自带背压和阻塞语义，天然匹配生产者-消费者模型。

DemuxThread → videoPacketQ (100) → VideoDecodeThread → videoFrameQ (15) → VideoRenderer
DemuxThread → audioPacketQ (400) → AudioDecodeThread → QAudioOutput → setAudioClock AVSync ← query VideoRenderer

关键场景时序

▶ 播放

1. PlayerController::play() 检查 playing_ 标志
2. start DemuxThread → start VideoDecodeThread → start AudioDecodeThread
3. VideoRenderer::startRendering() 启动 1ms QTimer
4. AudioDecodeThread 启动 QAudioOutput → 开始输出 PCM
5. posTimer (100ms) 开始轮询 AVSync 发送位置更新

⏸ 暂停

1. PlayerController::pause() 设置 playing_ = false
2. AudioDecodeThread::setPaused(true) → QAudioOutput::suspend()
3. DemuxThread / VideoDecodeThread 继续运行但队列积压
4. posTimer 停止发送 positionChanged
5. 恢复时 resume QAudioOutput → playing_ = true

⏩ Seek

1. PlayerController::seek(s) → DemuxThread::seek(s) 原子存储
2. DemuxThread::run() 检测 seekTarget_ ≥ 0 → av_seek_frame
3. 清空 videoPacketQ + audioPacketQ
4. VideoDecodeThread::flush() + AudioDecodeThread::flush() 清编码器缓冲
5. VideoRenderer::flushPendingFrame() 丢弃暂存帧
6. 恢复读取，从新位置开始分发

⏹ 停止

1. PlayerController::stop() → stopAllThreads()
2. DemuxThread::stop() + wait() → 队列 abort() 唤醒所有阻塞
3. VideoDecodeThread::stop() + wait()
4. AudioDecodeThread::stop() + wait() → QAudioOutput::stop()
5. VideoRenderer::stopRendering()
6. 三队列 reset() 恢复正常状态
