② 三条 FrameQueue 的参数与角色

📦 videoPacketQ_ cap=100

60 pkts
类型：FrameQueue<AVPacket*>
生产：DemuxThread（av_packet_clone 推入）
消费：VideoDecodeThread（tryPop 取包）
满时：DemuxThread 阻塞，产生背压
容量选择：100 帧 × 平均包大小 ≈ 缓冲 ~4s

📦 audioPacketQ_ cap=400

300 pkts
类型：FrameQueue<AVPacket*>
生产：DemuxThread（音频包更小、更密）
消费：AudioDecodeThread
容量更大：音频帧约 23ms/帧，400帧≈9s 缓冲
保证音频连续性，防止爆音

🖼 videoFrameQ_ cap=15

5 frames
类型：FrameQueue<AVFrame*>
生产：VideoDecodeThread（av_frame_clone 推入）
消费：VideoRenderer::onTimer()（tryPop 非阻塞）
容量小：解码帧体积大（YUV420P 1080p≈3MB/帧）
15帧 × 3MB = 约 45MB 内存上限

③ open() 初始化流程——数据从文件流向各组件

stop() + reset() — 清除旧状态
重复打开文件时保证状态干净。stopAllThreads() 等待所有线程退出，三条队列 reset() 恢复可用。
stop();
videoPacketQ_.reset(); audioPacketQ_.reset(); videoFrameQ_.reset();
demux_.open() — 打开文件，绑定包队列
DemuxThread 调用 avformat_open_input + avformat_find_stream_info，识别视频/音频流索引，绑定两条包队列（但还未 start()）。
if (!demux_.open(path, &videoPacketQ_, &audioPacketQ_)) return false;
读取流参数 — AVFormatContext → codecpar + time_base
从已打开的 AVFormatContext 取视频/音频流，获得分辨率、codec id、时间基等参数，这些将传给解码器和渲染器。
AVFormatContext* fmt = demux_.formatContext(); // 已打开的格式上下文
int vi = demux_.videoStreamIdx(); // 视频流索引，-1=无视频
int ai = demux_.audioStreamIdx(); // 音频流索引，-1=无音频
AVCodecParameters* vp = fmt->streams[vi]->codecpar; // 含 width/height/codec_id
AVRational vtb = fmt->streams[vi]->time_base; // pts 单位换算用
videoDec_.init() — 打开视频解码器，绑定队列
avcodec_find_decoder + avcodec_open2，输入队列绑 videoPacketQ_，输出队列绑 videoFrameQ_。
videoDec_.init(vp); // 打开 H.264 解码器
videoDec_.setInputQueue(&videoPacketQ_);
videoDec_.setOutputQueue(&videoFrameQ_);
renderer_->init() — 渲染器绑定分辨率 + 时钟 + 帧队列
此时才知道视频实际分辨率（来自 codecpar），创建 SwsContext（YUV420P→RGB32）和 QImage 缓冲区。
renderer_->init(vp->width, vp->height, vtb, &sync_, &videoFrameQ_);
// → sws_getContext(w,h,YUV420P → w,h,RGB32)
// → currentFrame_ = QImage(w, h, Format_RGB32)
audioDec_.init() — 打开音频解码器 + swr + QAudioOutput
初始化 SwrContext（任意格式 → S16 Stereo 44100），创建 QAudioOutput 并获取 QIODevice*，绑定 AVSync 用于更新音频时钟。
audioDec_.init(ap, atb, &sync_);
// → swr_alloc + swr_init（→ S16 Stereo 44100）
// → QAudioOutput::start() 获取 QIODevice*
audioDec_.setInputQueue(&audioPacketQ_);
emit durationChanged() — 通知 UI 文件时长
demux_.duration() 返回微秒，除以 1000 转毫秒，MainWindow 用此值设置进度条最大值。
emit durationChanged(duration()); // us → ms

④ AVSync 音频主时钟——视频如何跟随音频

🕐 AVSync 内部结构

12.340s
audioClock
atomic<double> audioClock_
AudioDecodeThread 每解码一帧更新
读取无锁，视频随时可查
videoDelay(pts) 计算：
delay = pts − audioClock()
delay > 2.0s → 放回队列（音频未追上）
delay > 0 → msleep(delay×1000ms)
delay ≤ 0 → 立即渲染（视频落后追赶）
delay < -0.4s → 丢帧阈值（AVSync内部）

🎵 音频时钟更新路径

pts × av_q2d(timeBase) → 秒
sync_->setAudioClock(pts) ← 更新主时钟
swr_convert → PCM → QIODevice::write
VideoRenderer::onTimer()
tryPop(frame, 0) ← 非阻塞
pts = frame->pts × av_q2d(timeBase_)
delay = sync_->videoDelay(pts)
→ 等待 / 放回 / 立即渲染
音频时钟 12.34s

⚠ 为什么必须先停 DemuxThread？
若先停解码线程（消费方），DemuxThread 仍在 push 包，包队列满后 DemuxThread 阻塞在 push()， 永远等不到消费者，导致 wait() 超时、线程泄漏。
先停 DemuxThread → 包队列 abort → 解码线程 tryPop 返回 false → 自然退出。
