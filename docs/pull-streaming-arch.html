RambosPlayer — 拉流播放架构图

网络流拉取  ·  异步探测  ·  自动重连  ·  码率/帧率实时统计  ·  直播禁用 Seek

关键设计点：
• probeOpen() 静态方法：不访问 this 成员，可在任意 worker 线程调用，避免 avformat_open_input/avformat_find_stream_info 阻塞 UI（网络流 DNS/握手可能耗时数秒）。
• 异步 open + probeGeneration_：PlayerController::open() 启动 worker 线程探测，完成后通过 QThread::finished 信号回到主线程；probeGeneration_ 计数器丢弃被后续 open() 取代的过期结果。
• reconnect() 自动重连：网络流读取出错/断开时，2 秒间隔循环重试，分段检查 abort（100ms 步进），用户点"断开"可立即退出。
• NetworkState 状态机：Disconnected → Connecting → Connected → Reconnecting → Connected，通过 Qt 信号链 DemuxThread → PlayerController → MainWindow 更新状态栏。
• 直播模式限制：duration=0 时进度条禁用并显示 "LIVE"，seek 按钮不可用。
• 码率/帧率统计：DemuxThread::run() 内 1 秒窗口累计字节数和视频包数，emit statsUpdated(kbps, fps) 供 UI 实时显示。
